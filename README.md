# 👾 DR. SAP · Drone Room Scan Analysis Pipeline 

> This repo post-processes the raw point cloud generated by [VSLAM-UAV](https://github.com/sasankpanch/Volume-Measuring-VSLAM-UAV) to produce
> clean, accurate room maps. Visual SLAM output is inherently noisy - drift, sensor noise, and moving objects introduce outliers and redundant
> points that distort downstream measurements. This pipeline applies a sequence of filtering and clustering steps to turn that raw scan
> into a reliable geometric representation of the room.

## ✨ Point Cloud Cleaning Pipeline

**Pipeline Stages**
- *Voxel Downsampling*: reduce point density by averaging points within a fixed 3D grid, cutting computation cost 
 for later stages without losing overall room geometry.
- *Statistical Outlier Removal*: removes points whose distance to their neighbors deviates significantly from the
 local average; filtering sparse, isolated noise from SLAM drift.
- *Radius Outlier Removal*: removes points with too few neighbors within a fixed radius, catching noise the statistical
 filter misses.
- *DBSCAN Clustering*: groups points into spatially coherent clusters, separating meaningful structure (walls, floor, objects) 
 from residual noise and isolating relevant clusters for measurement.
- *RANSAC*: fits geometric primitives (e.g., planes for walls/floor) to the cleaned clusters, robust to any outliers
 that survive the earlier filtering stages.
- *Normalization*: aligns and scales the resulting point cloud into a consistent coordinate frame, enabling accurate, 
 repeatable room measurements across scans.

Together, these steps convert a noisy VSLAM point cloud into a clean, measurable 3D reconstruction of the mapped space.
